Servidor

Mis conocimientos en back-end se dividen en el entorno de programación Node.js y el lenguaje PHP. Ambos sistemas difieren entre si en muchos aspectos con ventajas para cada uno de ellos.

El primer lenguaje que aprendí ha sido Node.js. Se trata de un entorno de programación basado en JavaScript en el lado del servidor, que provoca que todo el código desarrollado sea asíncrono. La gran ventaja de esto es su bajo consumo de memoria y su rapidez de ejecución con lo que se obtiene un gran rendimiento en entornos de estrés.

El asincronismo de Node.js junto con la API Socket.IO facilitan de manera considerable el desarrollo de aplicaciones en tiempo real, característica aprovechada en el proyecto de final de carrera de la Uvigo.

Mis referencias a la hora de aprender a programar en Node.js han sido su documentación oficial y el libro Node.js de George Ornbo.

Con respecto a PHP, los motivos por los que decidí aprenderlo junto con el framework Symfony han sido su demanda profesional y su estilo de programación completamente diferente a Node.js, debido a que no se trata de un sistema asíncrono, siendo más parecido a lenguajes como C o Java, de los que ya tenía experiencia a lo largo de los cursos y las titulaciones.

El framework Symfony me ha parecido interesante para el desarrollo modular de la aplicación, utilizandolo en el proyecto del MISTIC como entorno de programación.

Las referencias que he utilizado para aprender PHP han sido el libro Curso de programaión Web de Scott McCracken y su documentación oficial. Con respecto a Symfony solamente he utilizado su libro oficial.

Bases de datos

Con respecto a las bases de datos también tengo experiencia en dos sistemas completamente diferentes. Por un lado, he utilizado un lenguaje No-SQL en el proyecto de la Uvigo, concretamente MongoDB. Aunque debido al uso de Mongoose para facilitar la organización de la base de datos no me ha permitido desarrollarla de una manera completamente No-SQL, he entendido el tipo de bases de datos y las posibilidades que ofrece, adaptándose perfectamente al entorno Node.js debido al uso de objetos JSON para el almacenamiento de datos.

Por otro lado, con respecto a las bases de datos SQL la que más he utilizado ha sido MySQL que es con la que más experiencia tengo, aunque también he utilizado Oracle a lo largo del MISTIC.

Sin lugar a dudas, las bases de datos son uno de mis puntos favoritos a la hora de desarrollar una aplicación web, llevándome a escoger como tema principal para el proyecto de fin de máster los ataques de inyección SQL.

La documentación utilizada para aprender acerca de MongoDB ha sido su documentación oficial y para SQL, a parte de los materiales del curso realizado en la Uvigo utilizo como referencia el libro SQL de Francisco Charte Ojeda.

Front-end

Mi experiencia en Front-end es más corta que con las tecnologías en el lado del servidor. Mis proyectos han carecido de una presentación visual atractiva por falta de conocimientos en maquetación web. Por ese motivo he decidido realizar el conjunto de cursos en desarrollo Front-end de Platzi, que aunque aún no he realizado el examen de certificación, me ha servido para realizar este portafolio.

Antes de realizar los cursos había aprendido por mi cuenta a utilizar jQuery, desde el proyecto de la Uvigo, para una gestión más sencilla de JavaScript debido a la importancia que toma el lenguaje a lo largo de todo el proyecto. Para aprender a utilizarlo he utilizado su documentación oficial.

Con respecto a CSS he utilizado Bootstrap desde el proyecto del MISTIC. En este caso, yo era el responsable tanto del desarrollo Front-end como el desarrollo Back-end. Bootstrap me ha ayudado a realizar una presentación mucho más atractiva gracias a sus diseños pre-definidos pudiendo añadirlos a la página de una manera sencilla. Al igual que con jQuery, he utilizado la documentación oficial para aprender a utilizar dicho framework.

Smartphone

Mi experiencia e desarrollo de Apps para smartphones no es muy extensa. Aunque he realizado el curso de desarrollo android en la Uvigo, nunca he desarrollado ninguna App en código nativo por mi cuenta.

En el proyecto de la Uvigo he desarrollado una aplicación móvil. Debido a la necesidad de utilizar websockects tanto en la aplicación web como en la aplicación móvil el desarrollo en Java se atojaba complicado, o por lo menos más dificil que el desarrollo con Cordova.

Gracias a Cordova he reutilizado gran parte del código desarrollado en la aplicación web en la aplicación móvil. Este sistema permite realizar Apps móviles con código web (HTML, CSS y JS) y exportarlas a casi cualquier entorno móvil (Android, IOS, Windows Phone, etc.). Para aprender a utilizar este sistema tan solo he tulizado su documentación oficial.

Seguridad

El MISTIC me ha dado la base para el desarrollo de aplicaciones seguras y una de las referencias para dicho desarrollo es OWASP. Dentro de esta página puedes encontrar multitud de posibles ataques y cómo solucionarlos, siendo de gran utilidad a la hora de desarrollar una aplicación web.

Otra herramienta que utilizo al terminar una aplicación es Kali Linux. Aunque no he realizado la especialidad de pentesting y auditoría en el MISTIC he aprendido a utilizar esta distribución gracias al libro Pentesting con Kali. Es una distribución muy sencilla de utilizar con las herramientas de pentesting más avanzadas del momento. Desde ella se pueden realizar los pasos de recogida de información, análisis y explotación de una aplicación web. Esto me ayuda a detectar problemas antes del desplegar cualquier aplicación a un entorno de desarrollo.
